MNSwiftKit

一套面向 iOS 日常工程的 Swift 组件库：从基础控制器、网络请求、Toast、刷新、数据库，到资源选择、媒体导出、播放器、表情键盘和分页容器，支持按模块集成，也可以一次引入完整能力。

安装方式

完整库适合快速接入；子模块适合降低依赖体积。CocoaPods 子模块名与源码目录一致，SPM Product 使用 MN 前缀。

CocoaPods 完整安装

默认 Full 聚合所有子模块。

pod 'MNSwiftKit'

CocoaPods 按需安装

只引入业务需要的模块。

pod 'MNSwiftKit/Base'
pod 'MNSwiftKit/Toast'
pod 'MNSwiftKit/AssetPicker'

Swift Package Manager

Xcode 添加仓库后选择 Product。

.product(name: "MNToast", package: "MNSwiftKit")
.product(name: "MNNetworking", package: "MNSwiftKit")

工程能力

库的设计重点不是单个控件，而是把高频 iOS 开发链路拆成稳定、可复用的模块。

UI

UIKit 组件体系

基础控制器、导航栏、标签栏、分页容器、Alert、Popover、空数据页、编辑态工具条。

AV

媒体工作流

相册选择、资源浏览、GIF/LivePhoto/视频处理、AVAsset 导出、音视频播放。

NET

网络与数据

请求序列化、响应解析、上传下载、断点数据库、SQLite 模型映射和 StoreKit 购买流程。

KIT

基础工具扩展

命名空间、屏幕尺寸定义、常用扩展、权限、扫码、二维码、触感、Keychain/认证相关工具。

Base

应用基础架构模块，统一控制器生命周期、导航栏、标签栏、列表加载、WebView 和网页支付流程。

核心 API

  • MNBaseViewController通用 UIViewController 基类，提供初始化钩子、内容视图、加载状态和导航状态处理。
  • MNExtendViewController在 Base 基础上扩展导航栏支持，适合业务控制器继承。
  • MNNavigationController隐藏系统导航栏并接入自定义转场能力的导航控制器。
  • MNNavigationBar / MNNavigationBarDelegate自定义导航栏、标题、按钮及交互代理。
  • MNTabBarController / MNTabBar / MNTabBarItem自定义 TabBar 容器、角标和重复点击回调。
  • MNListViewController集成列表视图、刷新、加载更多与空数据状态的列表基类。
  • MNWebViewController封装 WKWebView，支持 URL/HTML 加载、进度条、脚本桥、前进后退与关闭逻辑。
  • MNWebPayController / MNWebPayment为微信、支付宝等网页支付场景提供控制器和回调处理。
let controller = MNWebViewController(url: URL(string: "https://example.com")!)
controller.reloadWhenAppear = true
controller.allowsUpdateTitle = true
navigationController?.pushViewController(controller, animated: true)
安装方式
Podpod 'MNSwiftKit/Base'
SPM.product(name: "MNBase", package: "MNSwiftKit")

依赖 Toast、Refresh、Request、Definition、Extension、EmptyView、NameSpace、Transitioning、CollectionLayout。WebView 加载非 HTTPS 内容时需按业务配置 ATS。

Components

一组可直接落地的 UIKit 基础组件，适合替代项目中重复实现的弹窗、按钮、选择器和输入控件。

核心 API

  • MNAlertView支持标题、消息、文本框、动作按钮和通知关闭的自定义 Alert。
  • MNAlertAction / MNAlertField描述 Alert 动作和输入框配置。
  • MNPopoverView支持箭头方向、动画类型、分割线约束和 item 布局的弹出菜单。
  • MNButton自定义 UIControl 按钮，便于处理图片、标题和状态展示。
  • MNSegmentedControl分段选择控件。
  • MNDatePicker日期选择视图。
  • MNTextView / MNTextViewUpdater带更新回调的文本视图。
  • MNNumberKeyboard数字键盘和代理事件。
  • MNScanView扫码场景中的扫描框 UI。
  • MNActivityIndicatorView深色/浅色活动指示器。
  • MNSecureView安全输入或隐私保护容器。
let alert = MNAlertView(title: "删除记录", message: "此操作不可撤销", preferredStyle: .alert)
alert.addAction(title: "取消", style: .cancel)
alert.addAction(title: "删除", style: .destructive) { _ in
    // handle delete
}
alert.show()
安装方式
Podpod 'MNSwiftKit/Components'
SPM.product(name: "MNComponents", package: "MNSwiftKit")

依赖 Extension、NameSpace。使用扫码 UI 搭配真实相机采集时，需要配置相机权限说明。

AssetPicker

面向相册选择业务的完整模块，覆盖多选、预览、GIF/LivePhoto、视频导出、裁剪和相册分页加载。

核心 API

  • MNAssetPickerUINavigationController 子类，提供 present 入口和选择/取消回调。
  • MNAssetPickerOptions选择器配置：最大/最小数量、照片/视频/GIF/LivePhoto、混选、预览、导出、列数、主题色、深浅模式。
  • MNAssetPickerDelegate选择完成与取消事件代理。
  • MNAsset统一描述 PHAsset、封面、文件 URL、资源类型、更新回调和浏览协议。
  • MNAssetAlbum相册模型。
  • MNAssetHelper相册权限、资源读取、保存和导出相关工具入口。
  • MNTailorViewController图片或视频裁剪控制器，配合 MNTailorView、MNTailorHandler 使用。
  • MNPickError选择、读取和导出过程中的错误枚举。
let options = MNAssetPickerOptions()
options.maxPickingCount = 9
options.allowsPreview = true
options.allowsPickingVideo = true
options.allowsExportVideo = true

MNAssetPicker(options: options).present(in: self) { assets in
    // use selected MNAsset values
}
安装方式
Podpod 'MNSwiftKit/AssetPicker'
SPM.product(name: "MNAssetPicker", package: "MNSwiftKit")

需要 NSPhotoLibraryUsageDescription；保存到相册需 NSPhotoLibraryAddUsageDescription。若接入拍摄或视频录制，还需相机、麦克风权限。

AssetBrowser

资源浏览器模块，用于全屏预览图片、GIF、LivePhoto 和视频，并支持拖拽关闭、缩放、保存、分享和自定义数据源。

核心 API

  • MNAssetBrowser主浏览视图，接收 MNAssetBrowseSupported 数组并调用 present 展示。
  • MNAssetBrowseSupported资源数据协议，描述 identifier、contents、type、cover 和进度。
  • MNAssetBrowseDelegate监听浏览器事件、页面切换、按钮触发和状态变化。
  • MNAssetType资源类型：photo、gif、livePhoto、video。
  • MNAssetBrowserCell浏览单元格，负责封面、内容、播放视图和缩放容器。
  • MNAssetScrollView图片缩放和拖拽容器。
let browser = MNAssetBrowser(assets: assets)
browser.autoPlay = true
browser.dragToDismiss = true
browser.rightBarItemEvent = .share
browser.present(in: view, from: 0)
安装方式
Podpod 'MNSwiftKit/AssetBrowser'
SPM.product(name: "MNAssetBrowser", package: "MNSwiftKit")

依赖 Slider、Player、Definition、Extension、AnimatedImage。浏览 PHAsset 或保存资源时需相册读取/写入权限。

Networking

URLSession 网络层封装，关注请求构造、参数编码、响应解析、安全策略、可达性和统一错误信息。

核心 API

  • MNNetworkSession会话对象，承载 URLSessionTask、上传/下载进度和完成回调。
  • MNNetworkProxy任务代理，管理 fileURL、parser、Progress、上传下载位置与完成处理。
  • MNNetworkSerializer根据 MNNetworkMethod、参数和内容类型生成 URLRequest。
  • MNNetworkParser解析 Data、JSON、String、文件下载和状态码。
  • MNNetworkParam请求参数结构。
  • MNNetworkContentType请求/响应 Content-Type 枚举。
  • MNNetworkReachability网络状态监听与通知。
  • MNNetworkSecurityPolicy证书校验和安全策略。
  • MNNetworkError统一错误枚举，内置编码、解析、文件、状态码等错误常量。
let serializer = MNNetworkSerializer()
let parser = MNNetworkParser()
// 与 MNNetworkSession / MNNetworkProxy 组合，用于请求、上传、下载和解析结果。
安装方式
Podpod 'MNSwiftKit/Networking'
SPM.product(name: "MNNetworking", package: "MNSwiftKit")

包含 MNNetworkReason.json 错误原因资源。访问 HTTP 或自签证书服务时，需按 App 安全策略配置 ATS/证书校验。

Request

在 Networking 之上的业务请求层，将请求生命周期抽象成可继承的 Request 类型，适合分页、上传、下载和文件数据业务。

核心 API

  • MNRequest请求基类，定义请求生命周期、结果处理和状态。
  • MNDataRequest数据请求基类；MNPagingRequestSupported 为分页请求提供 page/limit 等约定。
  • MNUploadRequest上传请求封装。
  • MNDownloadRequest下载请求封装。
  • MNFileDataRequest文件数据请求。
  • MNRequestResult统一结果对象。
  • MNRequestManager请求管理器，负责队列、恢复和取消。
  • MNRequestDatabase下载/上传等请求本地记录，依赖 SQLite。
  • MNUploadAssistant辅助构造上传数据。
安装方式
Podpod 'MNSwiftKit/Request'
SPM.product(name: "MNRequest", package: "MNSwiftKit")

依赖 Networking 和 sqlite3。下载到相册或读取本地安全域文件时，需按实际业务补充系统权限或安全域访问处理。

Toast

统一反馈提示模块，适合加载、成功、失败、信息、进度和可取消任务提示。

核心 API

  • MNToast主视图，提供 show(_:in:at:status:progress:cancellable:delay:close:) 和显示时长计算。
  • MNToastBuilderToast 内容构建协议。
  • MNToastCancelSupported声明可取消行为。
  • MNToastAnimationSupported声明自定义动画能力。
  • MNToastProgressSupported声明进度展示。
  • MNToastCloseSupported声明关闭按钮能力。
  • MNMsgToast / MNInfoToast / MNSuccessToast / MNErrorToast文本、信息、成功和错误提示构建器。
  • MNProgressToast / MNActivityToast / MNRotationToast进度、活动指示和旋转类型提示。
  • UIView+MNToast为 UIView 提供命名空间提示入口。
安装方式
Podpod 'MNSwiftKit/Toast'
SPM.product(name: "MNToast", package: "MNSwiftKit")

依赖 NameSpace，包含深浅遮罩和图标资源。无额外系统权限。

Refresh

UIScrollView 刷新控件模块，用于下拉刷新、上拉加载、状态展示和滚动视图扩展。

核心 API

  • MNRefreshComponent刷新组件基类，定义状态、回调、布局和触发逻辑。
  • MNRefreshHeader下拉刷新基类。
  • MNRefreshStateHeader带状态文本的下拉刷新头。
  • MNRefreshFooter上拉加载基类。
  • MNRefreshStateFooter带状态文本的加载更多尾部。
  • UIScrollViewRefreshing为滚动视图挂载刷新组件的扩展入口。
安装方式
Podpod 'MNSwiftKit/Refresh'
SPM.product(name: "MNRefresh", package: "MNSwiftKit")

Player

基于 AVPlayer 的播放模块，支持单资源和队列播放、进度、缓冲、播放速率、音量与 seek。

核心 API

  • MNPlayer播放核心。公开 play、pause、forward、playNext、seek、队列增删改查等方法。
  • MNPlayerDelegate监听播放状态、时间、缓冲、错误和结束事件。
  • MNPlayView视频渲染视图和交互代理。
  • MNPlayError播放错误枚举。
  • PlaySoundSupported声音播放支持协议。
let player = MNPlayer()
player.append(URL(fileURLWithPath: path))
player.prepare()
player.play()
player.seek(progress: 0.5)
安装方式
Podpod 'MNSwiftKit/Player'
SPM.product(name: "MNPlayer", package: "MNSwiftKit")

播放本地/网络媒体无需权限；后台播放需开启 UIBackgroundModes audio，录音场景另配麦克风权限。

Database

轻量 SQLite 封装，支持通过模型协议或字段字典完成建表、插入、更新、删除、查询和聚合计算。

核心 API

  • MNDatabase数据库入口。提供 open、close、create、insert、update、delete、selectRows、selectCount、calculate。
  • MNDatabase.default默认数据库实例。
  • MNTableRowInitializable模型与表行之间的映射协议。
  • MNTableColumn字段描述，包含 name、type、primary、nullable 等配置。
  • MNTableColumnSupported / MNTableColumnAssignment字段集合和赋值协议。
let database = MNDatabase.default
database.open()
database.create(table: "users", using: User.self)
database.insert(into: "users", using: user)
let rows = database.selectRows(from: "users", type: User.self)
安装方式
Podpod 'MNSwiftKit/Database'
SPM.product(name: "MNDatabase", package: "MNSwiftKit")

链接 sqlite3。无额外系统权限，注意自行处理用户敏感数据的加密与备份策略。

MediaExport

AVAsset 导出工具链，面向视频裁剪、音轨/视频轨控制、输出质量、时间范围和导出进度。

核心 API

  • MNAssetExportSession资产导出会话封装。
  • MNMetadataExportSupported元数据导出支持协议。
  • MNExportError导出错误枚举。
  • AVAsset/AVAssetTrack/AVMutableComposition 扩展媒体轨道、组合和导出辅助能力。
let session = MNMediaExportSession(asset: asset, outputURL: outputURL)
session.quality = .high
session.timeRange = timeRange
session.exportAsynchronously { progress in

    // handle result

Purchase

StoreKit 内购模块，提供购买、恢复购买、票据读取、票据校验、状态回调和通知。

核心 API

  • MNPurchaseManager.default默认购买管理器，负责成为交易观察者、发起购买、恢复和续接未完成任务。
  • startPurchasing(_:userInfo:status:completion:)根据 productId 发起购买。
  • startRestore(status:completion:)恢复购买。
  • startCheckout(status:completion:)发起票据校验。
  • MNPurchaseReceipt票据模型。
  • MNPurchaseResult购买或校验结果。
  • MNPurchaseRequest购买请求对象，暴露状态回调。
  • MNReceiptBase本地票据读取和存储基础。
  • MNPurchaseDidFinishNotification购买完成通知。
安装方式
Podpod 'MNSwiftKit/Purchase'
SPM.product(name: "MNPurchase", package: "MNSwiftKit")

依赖 StoreKit 和 sqlite3。需在 App Store Connect 配置内购商品，并在工程能力中启用 In-App Purchase。

EmoticonKeyboard

表情键盘模块，内置表情资源、分包管理、输入视图、富文本附件和输入控件适配。

核心 API

  • MNEmoticonKeyboard表情键盘主视图和代理事件。
  • MNEmoticonKeyboardOptions键盘样式、布局和行为配置。
  • MNEmoticonManager表情包增删改查，并发布 added、removed、changed 通知。
  • MNEmoticonPacket表情包模型。
  • MNEmoticon单个表情模型。
  • MNEmoticonAttachment富文本附件，用于把表情嵌入 attributed string。
  • MNEmoticonBackedString表情文本和附件之间的桥接。
  • MNEmoticonInputSupportedUITextView/UITextField 等输入控件适配协议。
  • NSAttributedString+MNEmoticonSupported富文本表情解析和生成。
安装方式
Podpod 'MNSwiftKit/EmoticonKeyboard'
SPM.product(name: "MNEmoticonKeyboard", package: "MNSwiftKit")

依赖 Definition、Extension、NameSpace、PageControl、AnimatedImage，包含内置表情资源。无额外系统权限。

安装提示

这些模块也可以单独安装。作为上层功能依赖时，CocoaPods 与 SPM 都会随业务入口模块自动解析依赖；直接使用某个轻量模块时按对应条目引入即可。

权限配置提醒

如果使用 AssetPicker、Utility、Player 等涉及相册、相机、麦克风、Face ID、IDFA 或后台音频的能力，请在宿主 App 的 Info.plist 中补充对应 Usage Description。只按需安装模块时，也只需要配置实际调用能力所需的权限。
